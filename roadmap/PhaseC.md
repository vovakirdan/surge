# Что делаем в Phase C (байткод и .sbc)

## Цели (итог фазы)

* Минимальная **стековая VM** (интерпретатор) с устойчивым fetch–decode–execute циклом.
* Компилятор backend → **байткод** по стабильному **ISA v0**.
* Двоичный формат **.sbc v0** (заголовок + секции).
* **Дизассемблер**: читает .sbc и печатает человекочитаемый листинг.
* Набор е2е-тестов: .sg → .sbc → дизассемблер/VM → корректные результаты.

## Объем первой версии (что точно входит)

* Типы значений в VM: `i64`, `f64`, `bool`, `string*`, `array*` (указатели на рантайм-объекты).
* Функции, локальные переменные, глобалы, базовый контроль потока.
* Массивы (создание, длина, get/set).
* Никакой параллельщины / каналов / реактивности в рантайме (пока). Синтаксис есть, кодоген — позже.
* Ссылки/own/deref в рантайме пока **не исполняем**: это останется на семантике до появления нормальной памяти. (Байткод для них в v0 не нужен.)

## Архитектура VM (интерпретатор)

* **Value** (универсальное значение):

  ```c
  enum ValTag { V_I64, V_F64, V_BOOL, V_STR, V_ARR, V_NULL };
  struct Value { uint8_t tag; union { int64_t i; double f; bool b; void *ptr; } u; };
  ```
* **Стек**: массив `Value` фикс./растущий (зададим лимит, проверка переполнения).
* **Кадр вызова**:

  ```
  [ ... caller stack ... ][locals][temporaries...]
                           ^bp                 ^sp
  frame: { return_ip, prev_bp, func_index, base (bp) }
  ```
* **Локалы**: индексируются 0..N-1 (*N* известно из таблицы функций).
* **Глобалы**: простая таблица `Value gvars[]` + init-функция `__init__` для присвоений на уровне модуля.
* **Память строк/массивов**: минимальный рантайм-хип (C-аллокатор). v0: без GC, без RC; живут до завершения VM. (Ок для тестов; настоящую жизнь/RC добавим в фазе memory/runtime.)

## ISA v0 (опкоды и операнды)

Кодировка: `OP (1 байт) + операнды в little-endian`. Смещения — относительные к `ip` (signed 32-bit).

**Константы / литералы**

* `PUSH_I64 <i64>`
* `PUSH_F64 <f64>`
* `PUSH_BOOL <u8>` (0/1)
* `PUSH_STR <u32 const_idx>` (строки в констант-пуле)
* `PUSH_NULL`

**Локалы / глобалы**

* `LOAD  <u16 slot>`
* `STORE <u16 slot>`
* `GLOAD  <u16 gslot>`
* `GSTORE <u16 gslot>`

**Арифметика (динамическая проверка тега, без авто-каста)**

* `ADD`, `SUB`, `MUL`, `DIV`, `REM`  → int or float only (ошибка в рантайме иначе)

**Сравнения / логика**

* `CMP_EQ, CMP_NE, CMP_LT, CMP_LE, CMP_GT, CMP_GE` → пушат `bool`
  (числа попарно одного вида; для строк/массивов — только `EQ/NE` по указателю в v0)
* `JMP <s32>`
* `JMP_IF_TRUE <s32>` / `JMP_IF_FALSE <s32>` (попают `bool`)

**Вызовы**

* `CALL <u16 func_idx> <u8 argc>`  (аргументы заранее на стеке)
* `RET` (если функция имеет возвращаемое значение — один `Value` остаётся на вершине, иначе ничего; ответственность кодогенератора)

**Массивы**

* Литерал «desugared»: `push elem ...; ARR_NEW <u32 count>`
* `ARR_NEW <u32 count>` — выталкивает `count` значений, создаёт массив, пушит `array*`
* `ARR_LEN` — попает `array*`, пушит `i64`
* `ARR_GET` — попает `idx(i64)`, `array*`, пушит элемент (копия)
* `ARR_SET` — попает `value`, `idx(i64)`, `array*`, записывает, **ничего не пушит**

  > индексы проверяем, иначе TRAP

**Спец**

* `TRAP <u16 code>` — аварийная остановка с кодом
* `HALT` — завершение исполнения (обычно в конце `main`)

> Примечание: в v0 не делаем типизированные версии `ADD_I64/ADD_F64` — семантика уже гарантирует типы, VM лишь проверяет и аварийно падает при нарушении.

## Формат `.sbc v0`

Все поля little-endian. Выравнивание до 4 байт.

**Header (фикс. размер)**

```
u32 magic = 'SBC0'    // 0x30 0x43 0x42 0x53
u16 ver_major, ver_minor        // например 0,1
u16 flags                        // зарезервировано
u32 off_consts, len_consts
u32 off_funcs,  len_funcs
u32 off_code,   len_code
u32 off_strings,len_strings      // string blob (0-terminated, на которые указывают константы)
```

**Const Pool (vector)**

```
u32 count
repeat count:
  u8  tag  (C_I64=1/C_F64=2/C_BOOL=3/C_STR=4)
  pad(3)
  union { i64; f64; u8 bool + pad(7); u32 str_off + pad(4); }
```

**Function Table (vector)**

```
u32 count
repeat count:
  u32 name_str_off  // 0 если безымянная (например __init__)
  u16 n_params
  u16 n_locals      // локалов ИСКЛЮЧАЯ параметры (или включая? — фиксируем: исключая)
  u8  has_ret       // 0/1 (детализация типа нам не нужна в VM v0)
  u8  reserved[3]
  u32 code_off
  u32 code_len
  u32 dbg_off       // 0 если нет (линейная карта опционально)
```

**Code Section**

* просто байт-код; каждая функция по `code_off/code_len`.

**Strings Section**

* конкатенация `'\0'`-терминированных строк.

**Globals**

* В v0 храним «глобальные переменные» как записи в таблице функций: исполняем `__init__` (func\_idx=0) перед `main`.

  * `GLOAD/GSTORE` работают с индексами (компилятор сам назначает).

## Компиляция из AST в байткод (backend)

* Таблицы окружений:

  * `locals[name] -> slot`
  * `globals[name] -> gslot`
  * `funcs[name] -> func_idx`
* Генерация выражений (в стек).
* Присваивание `name=expr` → `expr; STORE <slot>`.
* Блоки/области → просто размещение локалов (число слотов известно заранее из одноразового прохода по фну).
* `if`: `cond; JMP_IF_FALSE else; then; JMP end; else: ...; end:`
* `while`: `loop: cond; JMP_IF_FALSE end; body; JMP loop; end:`
* Литералы массивов: последовательность `PUSH_* ...; ARR_NEW <N>`
* Функции:

  * Все определения попадают в таблицу функций.
  * Параметры считаем локалами `slot=0..n_params-1`, локалы после.
  * `return expr` → `expr; RET`; `return;` → `RET` (без значения).
* Top-level:

  * `let` → генерируем в `__init__`: вычислить expr и `GSTORE gslot`.
  * `signal` — пока игнорируем (или запрещаем в bytecode-компиляторе), но таблицу имён можно резервировать.
  * `main` — обычная функция; `surge runbc` запускает `__init__` → `main`.

## Дизассемблер

* Печатает:

  1. заголовок/версии,
  2. const pool (индексы и значения),
  3. функции (индекс, имя, nparams/nlocals),
  4. листинг байткода по функциям: `<ip>: OP [operands]  ; комментарии с const/string`
* Дополнительно: опция `--hex` (сырой дамп), `--sym` (символьные имена рядом).

## Проблемы/решения (на что смотрим)

* **Эндианность**: фиксируем на little-endian; валидатор заголовка откажет, если хост big-endian (пока).
* **Совместимость**: `ver_major/minor` в заголовке, дизассемблер/VM проверяют.
* **Ошибки рантайма**: `TRAP` для bounds/type errors; человекочитаемые коды/сообщения.
* **Строки/массивы**: v0 — «утекают» до конца процесса VM → ок для тестов. План: позже введём RC/arena/GC.
* **Ссылки/own**: пока «только в семантике». Реальное разыменование/адреса — после memory model.
* **Параллель/сигналы**: не в v0 VM. Позже — как спец-вызовы в рантайме (`CALL @rt_par_map`, `CALL @rt_emit`), или как опкоды.

## Тест-план

* **Юнит** на каждый опкод (микропрограммы).
* **Е2Е**:

  * арифметика/логика/ветвления;
  * функции/рекурсия/локалы;
  * массивы: литерал → `ARR_NEW`, `GET/SET`, `LEN`;
  * глобалы: инициализация в `__init__`.
* **Golden**: `.sg → .sbc → disasm` сравниваем со снапшотом (стабильность ISA).
* Негативные: деление тип-ошибок на уровне VM (`ADD` по строкам), выход за пределы.

## Дорожная карта Phase C (порядок работ)

1. **opcodes.h** + табличка метаданных (имя, кол-во операндов, их типы).
2. **.sbc writer/reader** (пустышка + sanity-checks).
3. **VM** каркас (HALT/stack/frames/LOAD/STORE/CALL/RET).
4. Константы + PUSH\_\*; сравнения + ветвления.
5. Массивы/строки (рантайм-объекты).
6. **Дизассемблер**.
7. **Backend** для подмножества языка (выражения, присваивания, `if/while`, функции, массивы).
8. ****init** + main** конвенция.
9. Е2Е-тесты + golden дизассемблера.
10. Документация `BYTECODE.md`.

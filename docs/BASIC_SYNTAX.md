# BASIC_SYNTAX (Phase B snapshot)

## 0) Общие положения

* **Файлы**: текст UTF-8, допускается BOM.
* **Идентификаторы**: `[A-Za-z_][A-Za-z0-9_]*`.
* **Разделители**: пробелы/табуляции/переводы строк незначимы, кроме случаев, где они разделяют токены.
* **Комментарии**:
  * построчные: `// ... до конца строки`
  * блочные: `/* ... */` (вложенность не поддерживается).
* **Точка с запятой `;` обязательна** после большинства инструкций.

## 1) Литералы и базовые выражения

* Целые: `0`, `42`, `123456` (`int64`-диапазон).
* Вещественные: `3.14`, `2.5e-3`, `1e9` (`double`).
* Булевы: `true`, `false`.
* Строки: `"text"`, escape: `\"`, `\\`, `\n`, `\t`, ...
* Массивы: `[expr, expr, ...]` (в том числе пустой `[]`).

```sg
let a:int = 42;
let b:float = 2.5e-3;
let ok:bool = true;
let s:string = "hello\nworld";
let xs:int[] = [1, 2, a];
````

## 2) Типы (Phase B)

* Примитивы: `int`, `float`, `bool`, `string`.
* Массив: `T[]`.
* Канал: `channel<T>`.
* Ссылки: `&T` (borrow).
* Владение: `own T`.
* Алиасы: `type I = int;`.

Пример:

```sg
type I = int;
fn id(x:int) -> int { return x; }

let x:int = 1;
let y:float = 3.0;
let xs:int[] = [1,2,3];
let rx:&int = &x;
fn fch(ch:channel<int>) -> int { return 0; }
```

## 3) Выражения: приоритеты и операции

### Унарные

* `-expr` (отрицание для int/float)
* `!expr` (логическое not)
* `&expr` (address-of: только lvalue)
* `*expr` (deref: только `&T`)

### Бинарные (левоассоциативны)

1. `||`
2. `&&`
3. `==`, `!=`
4. `<`, `<=`, `>`, `>=`
5. `+`, `-`
6. `*`, `/`, `%`

### Постфикс

* `callee(args...)`
* `base[index]`
* Реактивная привязка: `lhs := rhs` (выражение, а не оператор присваивания).

## 4) Инструкции (statements)

* `let name[:Type] = expr;`
* `signal name = expr;`
* Присваивание: `name = expr;`
* Инструкция-выражение: `lhs := rhs;`
* Блок: `{ stmt* }`
* `if (cond) { ... } else { ... }`
* `while (cond) { ... }`
* `return;` или `return expr;`
* `import name;`
* `type Name = Alias;`

## 5) Функции

```sg
@pure fn name(param1:Type1, param2:Type2, ...) [-> RetType] {
  stmt*
}
```

* Параметры требуют тип.
* Возврат опционален; без `->` считается `void`.
* `@pure` помечает функцию как чистую (нельзя IO, глобалки, сигналы).

Пример:

```sg
@pure fn add(a:int, b:int) -> int {
  return a + b;
}
```

## 6) Параллельные конструкции

### `parallel map`

```sg
parallel map <callee_expr> <seq_expr>;
```

* `callee_expr` — идентификатор функции.
* `seq_expr` — выражение (часто массив).

```sg
parallel map add [1,2,3];
```

### `parallel reduce`

```sg
parallel reduce <seq_expr> with (acc:Type, v:Type) => <expr>;
```

* `seq_expr` — массив.
* `(acc, v)` — параметры с типами.
* Тело — одно выражение.
* В `@pure`-контексте только чистые операции.

```sg
parallel reduce [1,2,3] with (acc:int, v:int) => acc + v;
```

## 7) Обязательные `;`

После: `let`, `signal`, `name = expr`, `lhs := rhs`, `parallel ...`, `import ...`, а также выражений как инструкций.
Не нужны после `}`.

## 8) Чистота функций и сигналы

* `@pure` — запрещены вызовы impure-функций, доступ к сигналам.
* В `parallel map/reduce` и `:=` допустимы только `@pure` функции.

## 9) Диагностика

Формат ошибок:

```
file:line:col: error: message
   source line
   ^
```

## 10) Ограничения снапшота

* Нет лямбд вне reduce.
* Присваивание только в идентификаторы (не `arr[i]`).
* Нет модулей/файловой системы для `import`.
* Нет автоматической расстановки `;`.
* Нет вывода типов (обязательно указывать).

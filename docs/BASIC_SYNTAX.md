# BASIC\_SYNTAX (Phase A snapshot)

## 0) Общие положения

* **Файлы**: текст UTF-8, допускается BOM.
* **Идентификаторы**: `[A-Za-z_][A-Za-z0-9_]*`.
* **Разделители**: пробелы/табуляции/переводы строк незначимы, кроме случаев, где они разделяют токены.
* **Комментарии**:

  * построчные: `// ... до конца строки`
  * блочные: `/* ... */` (вложенность не поддерживается).
* **Точка с запятой `;` обязательна** после большинства инструкций (см. ниже).

## 1) Литералы и базовые выражения

Поддерживаются:

* Целые: `0`, `42`, `123456` (`int64`-диапазон).
* Вещественные: `3.14`, `2.5e-3`, `1e9` (`double`).
* Булевы: `true`, `false`.
* Строки: `"text"`, допускаются escape-последовательности (`\"`, `\\`, `\n`, `\t`, ...).
* Массивы (литералы): `[expr, expr, ...]` (пустой `[]` тоже валиден).

Пример:

```sg
let a:int = 42;
let b:float = 2.5e-3;
let ok:bool = true;
let s:string = "hello\nworld";
let xs:int[] = [1, 2, a];
```

## 2) Типы (MVP)

* Имена типов — **идентификаторы** (пока без пространства имён).
* Слайс-суффикс `[]` разрешён **один раз**: `int[]`, `float[]`, `T[]`.
* Аннотация типов опциональна в `let`, обязательна в параметрах функции.
* Возврат у функций задаётся `-> Type`.

Примеры:

```sg
let x:int = 1;       // с типом
let y = 3.14;        // без аннотации (тип дальше выведет семантика)
fn add(a:int, b:int) -> int { return a + b; }
fn take(xs:int[]) -> int { return xs[0]; }
```

## 3) Выражения: приоритеты и операции

### Унарные

* Отрицание: `-expr`
* Логическое «не»: `!expr`

### Бинарные (левоассоциативны)

От низшего к высшему приоритету:

1. `||`
2. `&&`
3. `==`, `!=`
4. `<`, `<=`, `>`, `>=`
5. `+`, `-`
6. `*`, `/`, `%`

### Постфикс

* Вызов: `callee(args...)`
* Индексация: `base[index]`
* **Реактивная привязка**: `lhs := rhs` — это **выражение** (у нас парсится на постфиксном уровне, т.е. плотнее бинарных операций).

Примеры:

```sg
let r = f(1, 2) + arr[0] * 2;
y := x * k + 1;   // выражение; как инструкция — см. ниже
```

## 4) Инструкции (statements)

* **`let`**: объявление переменной.

  ```sg
  let name[:Type] = expr;
  ```
* **`signal`**: объявление реактивного сигнала.

  ```sg
  signal k = 2;
  ```
* **Присваивание** (пока только в идентификатор):

  ```sg
  name = expr;
  // Пример:
  x = x + 1;
  ```
* **Инструкция-выражение**: любое выражение, за которым следует `;`
  (типично используется для `lhs := rhs;`):

  ```sg
  y := x * k + 1;
  ```
* **Блок**:

  ```sg
  { stmt* }
  ```
* **`if` / `else`**:

  ```sg
  if (cond) { ... } else { ... }
  ```
* **`while`**:

  ```sg
  while (cond) { ... }
  ```
* **`return`**:

  ```sg
  return;
  return expr;
  ```
* **`import`** (минимально): импорт по имени модуля (пока без файлов/путей)

  ```sg
  import math;
  ```

## 5) Функции

```sg
fn name(param1:Type1, param2:Type2, ...) [-> RetType] {
  stmt*
}
```

* Параметры требуют явного типа.
* Возврат опционален (без `->` считается `void` на уровне синтаксиса).
* Тело — блок `{ ... }`.

Пример:

```sg
fn add(a:int, b:int) -> int {
  return a + b;
}
```

## 6) Параллельные конструкции (MVP)

Две формы-инструкции (каждая завершается `;`):

### `parallel map`

```
parallel map <callee_expr> <seq_expr> ;
```

* `callee_expr` — **базовое** выражение (идентификатор или литерал-лямбда в будущем; сейчас — идентификатор/простая форма без постфиксов).
* `seq_expr` — любое выражение (часто — массив-литерал).
  Если сразу после `callee_expr` идёт `[`, это трактуется как **литерал массива** для `seq_expr`, а не как индексатор `callee_expr[...]`.

Примеры:

```sg
parallel map add [1,2,3];
parallel map inc arr;      // arr — обычная переменная-массив
```

### `parallel reduce`

```
parallel reduce <seq_expr> with (acc:Type, v:Type) => <expr> ;
```

* `with` обязателен.
* Пара из параметров: аккумулятор `acc` и элемент `v`.
* Тело — **одно выражение** после `=>`.

Пример:

```sg
parallel reduce [1,2,3] with (acc:int, v:int) => acc + v;
```

## 7) Вызовы, индексация, скобки

* Вызовы: `f(x, y, z)`
* Индексация: `xs[i]`
* Скобки группируют выражения: `(a + b) * c`

## 8) Обязательные `;`

Требуются после:

* `let`, `signal`
* присваивания `name = expr;`
* **любого** выражения как инструкции (включая `y := ...;`)
* `parallel map ...;`, `parallel reduce ...;`
* `import name;`

Не требуются после:

* закрытия блока `}` внутри `if/while/fn` (тело само по себе блок).

## 9) Ошибки/диагностика (формат на Phase A pt.3)

* Сообщения: `file:line:col: error: message`
* Будет добавлен контекст исходной строки с указателем `^` под началом проблемного токена.

## 10) Ограничения текущего снапшота

* Типы — только имена + опциональный суффикс `[]` (один раз).
* Присваивание — только `name = expr;` (нет присваивания в `arr[i]`).
* Нет лямбд вне `reduce`-формы; в `reduce` тело — **одно** выражение после `=>`.
* Нет модулей/путей/импортов файловой системы — только `import <ident>;` в AST.
* Нет автоматических `;` (АСИ); ставим `;` вручную.
* Операторы и приоритеты — фиксированы (как выше).

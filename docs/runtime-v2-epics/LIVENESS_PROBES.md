# Runtime V2 Liveness Probes

Status: Epic 1 Task 6 planning. This file is documentation only. It does not
change runtime behavior, task status, shared README links, or working notes.

## Scope

Runtime V2 changes to scheduler, wakeup, waiter, fd registry, channel, timer,
cancellation, or shutdown code need a concrete liveness proof. Passing `make
check` is not enough: the current `make check` path sets
`SURGE_SKIP_TIMEOUT_TESTS=1`, so timeout-sensitive MT and LLVM probes are skipped.

Use this file with `EVIDENCE_TEMPLATE.md`:

- record the exact probe command in `Commands/Checks`;
- record trace rows and snapshots in `Trace Counters/Liveness Proof`;
- record skipped probes with the exact blocker and owner;
- keep Sentrux evidence separate. Missing Sentrux rules remain a Runtime V2 code
  blocker under `SENTRUX_POLICY.md`.

Current baseline debt matters. `01-baseline-evidence.md` records that
`go test ./internal/vm -run 'MT|Async|Net|LLVM'` fails in this checkout, while
the skipped form used by `make check` passes. Runtime-code tasks may not treat
the broad focused command as green. The debt is accepted for Epic 2 start and
belongs to a later test/backend matrix epic, but new runtime tasks must still
separate their own regressions from the recorded failure classes.

## Mandatory Gate By Change Type

Use the narrowest row that covers the changed surface. If a change touches more
than one surface, run every matching row.

| Changed surface | Mandatory liveness evidence |
| --- | --- |
| Scheduler ready queues, worker sleep/wake, work stealing, or seeded mode | MT process timeout wrapper; scheduler source trace; parked-with-work invariant; SIGUSR1 snapshot if the process can run long enough to sample. |
| Wakeup placement or wake-before-park handling | MT process timeout wrapper; direct async channel wakeups; net wakeup and live SIGUSR1 trace; parked-with-work invariant. |
| Global waiter list, owner-local waiters, or waiter cleanup | Cancellation/join/timeout smoke; MT correctness channel fixture; waiter cleanup by owner and back reference. |
| Channel send/recv/close or channel wait queues | Direct async channel wakeups; sync channel fallback and compensation if compatibility path still exists; channel close and cancellation race matrix; native channel benchmark trace script. |
| Net waiters, fd registry, I/O thread, or accept ownership | Net wakeup and live SIGUSR1 trace; FD registry wake persistence; native net benchmark trace script. |
| Timer or timeout internals | Cancellation/join/timeout smoke; sync channel fallback timer test; timer, timeout, and shutdown liveness. |
| Cancellation, scope, join, select, or failfast internals | Cancellation/join/timeout smoke; MT correctness channel fixture; waiter cleanup by owner and back reference; cross-shard cancellation generation if remote messages are involved. |
| Runtime shutdown state, worker teardown, I/O teardown, or process lifecycle | Timer, timeout, and shutdown liveness; benchmark timeout wrappers; final process-leak check in task evidence. |
| Performance-sensitive runtime path | Matching functional liveness probes plus native benchmark trace scripts with timeout wrappers and current-checkout `SURGE` binary verification. |

## Closeout Rule

Runtime-code tasks cannot close with "watch for hangs" as the liveness proof.
They must name the probe, command, expected counter or invariant, actual result,
blocker status, and owner for any missing implementation. Docs-only tasks may
record missing probes without implementing them, but must not claim those probes
passed.

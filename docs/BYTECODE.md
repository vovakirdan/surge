# Surge Bytecode (ISA v0)

Этот документ описывает текущую реализацию байткода Surge (версия 0.1) и
сопутствующие инструменты. Формат и набор опкодов стабилизированы для фаз A–C и
используются компилятором `surgec`, дизассемблером `surge disasm` и рантаймом
VM (каркас).

## Общие сведения

* Вычислительная модель — стековая машина.
* Каждая инструкция кодируется как 1 байт опкода + операнды в little-endian
  без выравнивания между ними.
* Смещения переходов (`JMP*`) задаются как знаковое `s32` относительно конца
  текущей инструкции.
* Значения времени выполнения (`Value`) типизируются тегом и payload'ом:
  `i64`, `f64`, `bool`, `null`, `string*`, `array*`.

## Набор опкодов

### Литералы и константы

| Опкод          | Операнды            | Описание |
|----------------|---------------------|----------|
| `PUSH_I64`     | `i64`               | Пушит знаковое 64-битное значение. |
| `PUSH_F64`     | `f64`               | Пушит `double` (IEEE-754). |
| `PUSH_BOOL`    | `u8` (`0`/`1`)      | Пушит булево. |
| `PUSH_STR`     | `u32 const_idx`     | Берёт строку из пула констант. |
| `PUSH_NULL`    | —                   | Пушит sentinel `null`. |

### Локальные и глобальные переменные

| Опкод   | Операнды      | Семантика |
|---------|---------------|-----------|
| `LOAD`  | `u16 slot`    | Читает локальную ячейку. |
| `STORE` | `u16 slot`    | Записывает в локальную ячейку. |
| `GLOAD` | `u16 gslot`   | Читает глобальную переменную (зарезервировано). |
| `GSTORE`| `u16 gslot`   | Записывает глобальную переменную (зарезервировано). |

### Арифметика, унарные операции и преобразования

Все инструкции проверяют теги операндов во время выполнения и при несовместимых
типах вызывают `TRAP TYPE_ERROR`.

#### Унарные операции

| Опкод          | Операнды | Описание |
|----------------|----------|----------|
| `NEG_I64`      | —        | Снимает `i64`, пушит `-v` (int). |
| `NEG_F64`      | —        | Снимает `f64`, пушит `-v` (float). |
| `NOT_BOOL`     | —        | Снимает `bool`, пушит логическое отрицание. |

#### Преобразования типов

| Опкод          | Описание |
|----------------|----------|
| `I64_TO_F64`   | Конвертация `i64 → f64` (точная). |
| `F64_TO_I64`   | Конвертация `f64 → i64` с отсечением дробной части в сторону нуля. При `NaN` или выходе за диапазон `int64` — `TRAP TYPE_ERROR` (overflow/NaN). |

#### Целочисленная арифметика и сравнения

| Опкод                  | Описание |
|------------------------|----------|
| `ADD`, `SUB`, `MUL`, `DIV`, `REM` | Бинарные операции над `i64`. Деление/остаток при делителе `0` → `TRAP DIV_BY_ZERO`. |
| `CMP_EQ`, `CMP_NE`, `CMP_LT`, `CMP_LE`, `CMP_GT`, `CMP_GE` | Сравнение `i64`, результат `bool`. |

#### Вещественная арифметика и сравнения (`f64`)

| Опкод                  | Описание |
|------------------------|----------|
| `ADD_F64`, `SUB_F64`, `MUL_F64`, `DIV_F64`, `REM_F64` | Арифметика над `f64`. Деление на ноль возвращает ±∞ по IEEE-754, остаток вычисляется через `fmod` (NaN при делителе 0). |
| `CMP_EQ_F64`, `CMP_NE_F64`, `CMP_LT_F64`, `CMP_LE_F64`, `CMP_GT_F64`, `CMP_GE_F64` | Сравнение `f64`, соблюдаются правила IEEE для NaN (все отношения, кроме `!=`, дают `false`). |

#### Битовые операции (`i64`)

| Опкод          | Описание |
|----------------|----------|
| `AND_I64`, `OR_I64`, `XOR_I64` | Побитовые операции над `i64`. |
| `SHL_I64`, `SHR_I64`           | Сдвиги влево/вправо (`>>` логический). Сдвиг ≥64 вызывает `TRAP TYPE_ERROR`. |

### Управление потоком

| Опкод            | Операнды       | Описание |
|------------------|----------------|----------|
| `JMP`            | `s32 offset`   | Безусловный переход. |
| `JMP_IF_TRUE`    | `s32 offset`   | Снимает `bool`; прыжок если истина. |
| `JMP_IF_FALSE`   | `s32 offset`   | Снимает `bool`; прыжок если ложь. |
| `TRAP`           | `u16 code`     | Немедленный рантайм-ошибка. |
| `HALT`           | —              | Остановка VM. |

Поддерживаемые trap-коды: `UNREACHABLE`, `DIV_BY_ZERO`, `OUT_OF_BOUNDS`,
`BAD_CALL`, `TYPE_ERROR`. Дизассемблер отображает их человекочитаемые имена.

### Вызовы функций

| Опкод  | Операнды                    | Описание |
|--------|-----------------------------|----------|
| `CALL` | `u16 func_index, u8 argc`   | Параметры уже на стеке (top-first). |
| `RET`  | —                           | Возвращает верхнее значение или `null`. |

Таблица функций в `.sbc` задаёт арность и количество локальных переменных, что
используется VM при создании кадра.

### Работа с массивами

| Опкод     | Операнды        | Семантика |
|-----------|-----------------|-----------|
| `ARR_NEW` | `u32 count`     | Забирает `count` значений и создаёт массив. |
| `ARR_GET` | —               | Снимает `(array, index)` и пушит элемент. |
| `ARR_SET` | —               | Снимает `(array, index, value)`; side effect. |
| `ARR_LEN` | —               | (Зарезервировано; не генерируется кодогеном v0.) |

Компилятор десугарит литерал `[a, b, c]` в `PUSH*; ARR_NEW 3`. Реактивное
присваивание `xs[i] := v` компилируется в `ARR_SET`.

## Формат `.sbc v0`

Все поля little-endian; секции выровнены на 4 байта.

### Заголовок (`SbcHeader`)

```
u32 magic      // 'SBC0'
u16 ver_major  // 0
u16 ver_minor  // 1
u32 off_const, sz_const
u32 off_funcs, sz_funcs
u32 off_code,  sz_code
u32 global_count // number of global slots
```

### Пул констант

```
u32 count
repeat count:
  u8  kind              // 1=i64, 2=f64, 3=str
  payload aligned to 4
```

* `i64`/`f64` — 8 байт значения.
* `str` — `u32 len` + байты строки + padding до 4.

### Таблица функций

Каждая запись сериализуется структурой `SbcFuncDesc`:

```
u32 name_idx   // индекс строки в пуле
u16 arity      // количество параметров
u16 nlocals    // включая параметры + локалы
u32 code_off   // смещение в секции кода
u32 code_len   // длина в байтах
u32 flags      // зарезервировано
```

Порядок функций соответствует индексам, используемым в опкоде `CALL`.

### Секция кода

Простая конкатенация байткода всех функций. `code_off/code_len` задают диапазоны.

## Инструменты

* `surgec file.sg -o out.sbc` — компиляция из AST в бинарный формат.
* `surge disasm out.sbc` — человекочитаемый листинг (показывает смещения, имена
  функций и trap-коды).
* `tools/disasm_golden.sh` — регрессии на устойчивость листинга (см. золотые
  дампы в `tests/codegen/golden`).


## Известные ограничения

* VM пока не выполняет байткод end-to-end; реализация цикла запланирована на
  фазу D.
* Глобальные переменные (`GLOAD/GSTORE`) поддерживаются для верхнеуровневых
  `let`; компилятор генерирует функцию `__global_init_auto__`, которую VM
  вызывает перед пользовательским `__init` и `main`.
* Нет поддержки каналов, сигналов, ссылок/own-типа в рантайме.
* Отладочные метаданные (linemap) не сериализуются.
